# 07. ERC721 & Metadata

## Konsep
- NFT: unik per `tokenId`, metadata via `tokenURI`.
- Minting, safeTransferFrom, approval per token.
- Base URI vs per-token URI.

## Langkah Praktik
1) Buat `contracts/LearnNFT.sol`:
```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;
import "@openzeppelin/contracts/token/ERC721/extensions/ERC721URIStorage.sol";

contract LearnNFT is ERC721URIStorage {
    uint256 private _nextId = 1;
    constructor() ERC721("Learn NFT", "LNFT") {}

    function mint(string memory uri) external returns (uint256) {
        uint256 tokenId = _nextId++;
        _mint(msg.sender, tokenId);
        _setTokenURI(tokenId, uri);
        return tokenId;
    }
}
```
2) Tambah test `test/erc721.ts`: mint, cek owner, cek tokenURI, transfer.
3) (Opsional) Tambah baseURI ketimbang URI per token dan bandingkan gas.

## Selesai Ketika
NFT bisa di-mint dan diuji, dan kamu tahu kapan memakai baseURI vs URI spesifik.
